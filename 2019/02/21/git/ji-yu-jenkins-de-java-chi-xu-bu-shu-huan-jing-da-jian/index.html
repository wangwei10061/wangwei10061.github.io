Jenkins实现Java项目的自动部署，基于Jenkins + Gitlab CI环境搭建

1. 介绍

需求：
有Maven管理的Java项目JenkinsTest，要求对JenkinsTest的每一次版本提交触发Jenkins的一次自动构建，并自动部署war包到服务器。

测试环境如下：
VMWare虚拟机1（192.168.249.129），1C2G，部署Jenkins服务，通过Tomcat运行Jenkins.war的方式
VMWare虚拟机2（192.168.249.130），1C2G，部署Gitlab服务，同时作为Tomcat应用服务器（后续工程war包部署的服务器）

2. 安装Jenkins插件

为Jenkins安装Maven Integration plugin（提供maven支持）及Publish Over SSH(因为Jenkins服务器与应用服务器不是同一服务器，需要传送war包到应用服务器)插件。

3. 创建Jenkins Job

创建Jenkins Maven Project，绑定仓库地址：

在打包完成之后，需要将war包传送到应用服务器，并部署到tomcat webapps目录，重启tomcat，作配置如下：

具体每一项参数的意思可以参见输入框后的说明，其中deploy.sh的内容如下：

#默认变量值
TOMCAT_HOME="/root/apache-tomcat-8.5.38"
TOMCAT_PORT=48081
PROJECT="$1"
#参数检验./deploy.sh 
if [ $# -lt 1 ]; then
  echo "you must use like this : ./deploy.sh "
  exit
fi
#根据端口查找tomcatpid,可能有多个,循环中判断
tomcat_pid=`netstat -anp | grep $TOMCAT_PORT | awk '{printf $7}' | cut -d "/" -f 1`
echo "current :" $tomcat_pid
while [ -n "$tomcat_pid" ]
do
 sleep 5
 #进一步筛选
 tomcat_pid=`ps -ef | grep $tomcat_pid |grep $TOMCAT_HOME | grep -v 'grep\|tail\|more\|bash\|less'| awk '{print $2}'`
 echo "scan tomcat pid :" $tomcat_pid
 if [ -n "$tomcat_pid" ]; then
   echo "kill tomcat :" $tomcat_pid
   kill -9 $tomcat_pid
 fi
done

#备份路径
BAK_DIR=$HOME/war/bak/$PROJECT/`date +%Y%m%d`
mkdir -p $BAK_DIR
cp $TOMCAT_HOME/webapps/$PROJECT.war $BAK_DIR/$PROJECT_`date +%H%M%S`.war &> /dev/null

#publish project
echo "scan no tomcat pid,$PROJECT publishing"
rm -rf $TOMCAT_HOME/webapps/$PROJECT
rm -rf $TOMCAT_HOME/webapps/$PROJECT.war
cp $HOME/war/$PROJECT.war $TOMCAT_HOME/webapps/$PROJECT.war

#remove tmp
#rm -rf $HOME/war/$PROJECT.war
#start tomcat
$TOMCAT_HOME/bin/startup.sh
echo "tomcat is starting,please try to access $PROJECT conslone url"
